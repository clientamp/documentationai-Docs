LickityClick is designed to be a flexible platform that leverages industry-standard open-source components for core link routing and security.

## Shlink

LickityClick uses [Shlink](https://shlink.io) as its underlying link routing engine. Shlink is a self-hosted, open-source URL shortener with a robust REST API.

<Tabs>
  <Tab title="Link Routing" icon="route">
    All redirects are handled directly by Shlink via high-performance RoadRunner or Swoole backends.
  </Tab>
  <Tab title="API Connection" icon="code-2">
    LickityClick communicates with Shlink via the **Shlink REST API v3**, using an admin API key for full control over short links, visits, and domains.
  </Tab>
  <Tab title="Multi-Domain" icon="globe">
    Manage multiple short domains (e.g., `dslink.me`, `clientamp.link`) through a single LickityClick dashboard.
  </Tab>
</Tabs>

## Better Auth

Security and user management are powered by [Better Auth](https://better-auth.com). This provides a secure, self-hosted authentication layer without needing a third-party service.

- **Email/Password Authentication**: Default sign-in method.
- **Session Management**: Secure, cookie-based sessions with configurable expiry.
- **Drizzle Adapter**: All authentication data is stored in your PostgreSQL database via the Drizzle ORM.

## MaxMind GeoLite2

For world-class geo-location tracking, LickityClick integrates with the **MaxMind GeoLite2** database.

- **Accurate Tracking**: Map incoming IP addresses to countries and cities.
- **Self-Hosted**: The database file (.mmdb) lives on your server, ensuring high performance and no third-party tracking calls during redirects.
- **Weekly Updates**: Keep your geo-location data accurate by scheduling weekly updates to the `.mmdb` file.

<Callout kind="info" title="Configuration">
  Set the `GEOLITE_DB_PATH` environment variable to the path of your `GeoLite2-City.mmdb` file to enable geo-tracking.
</Callout>

## Redis Caching

LickityClick uses **Redis** to cache Shlink API responses and frequent analytics queries.

- **Faster Load Times**: Reduced database and API overhead for frequently accessed dashboard views.
- **High Concurrency**: Handles bursts of traffic efficiently by caching the most popular redirect destinations.

## PostgreSQL 17

A single PostgreSQL 17 instance powers both the LickityClick application database and the Shlink engine database.

- **Unified Infrastructure**: Simplified backups and maintenance.
- **Drizzle ORM**: Type-safe database queries and automated migrations for the LickityClick app schema.

<Callout kind="success" title="Self-Hosted Freedom">
  LickityClick gives you full control over your data. All integrations are self-hosted or use direct API connections to your own infrastructure.
</Callout>
