This guide will help you get LickityClick running on your local machine for development or on a production server using Docker.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js**: 22 or later
- **pnpm**: 9 or later
- **Docker**: For running Shlink, PostgreSQL, and Redis

## Development Setup

Follow these steps to set up LickityClick for local development.

<Steps>
  <Step title="Clone & Install" icon="git-branch">
    ```bash
    git clone git@github.com:clientamp/lickityclick.git
    cd lickityclick
    pnpm install
    ```
  </Step>
  <Step title="Start Infrastructure" icon="docker">
    LickityClick requires PostgreSQL, Shlink, and Redis. The dev stack runs these via Docker:
    ```bash
    docker compose -f docker-compose.dev.yml up -d
    ```
  </Step>
  <Step title="Environment Configuration" icon="settings">
    Copy the example environment file and configure your secrets:
    ```bash
    cp .env.example .env.local
    ```
    *Note: The default values in `.env.example` match the `docker-compose.dev.yml` configuration.*
  </Step>
  <Step title="Database Init" icon="database">
    Push the schema and seed the database with an initial organization and user:
    ```bash
    pnpm db:push && pnpm db:seed
    ```
  </Step>
  <Step title="Run" icon="zap">
    Start the Next.js development server:
    ```bash
    pnpm dev
    ```
    Open [http://localhost:3000](http://localhost:3000) and log in with `dave@daveswift.com` (password: `changeme123`).
  </Step>
</Steps>

## Production Deployment

LickityClick is designed to be deployed as a suite of four Docker containers behind a reverse proxy like Caddy.

<Steps>
  <Step title="Server Setup" icon="server">
    Prepare your server (e.g., DS-1) and create a directory for the project:
    ```bash
    mkdir -p /opt/lickityclick
    cd /opt/lickityclick
    ```
  </Step>
  <Step title="Environment Variables" icon="key">
    Create a `.env` file with your production credentials. At a minimum, you must set:
    ```env
    BETTER_AUTH_SECRET=your-random-secret
    SHLINK_ADMIN_API_KEY=your-shlink-key
    DATABASE_URL=postgresql://app:password@db:5432/app
    ```
  </Step>
  <Step title="Start Everything" icon="rocket">
    Pull the latest images and start the containers:
    ```bash
    docker compose up -d
    ```
  </Step>
  <Step title="GeoLite2 Setup" icon="globe">
    For geo-location tracking, you need to download the MaxMind GeoLite2 database:
    ```bash
    # Create geodata volume directory
    mkdir -p geodata
    # Download GeoLite2-City.mmdb into ./geodata/
    # (See Deployment docs for download command with license key)
    docker compose restart app
    ```
  </Step>
</Steps>

<Callout kind="info" title="Caddy Configuration">
  LickityClick requires Caddy to handle `On-Demand TLS` for custom short-link domains. Ensure your `Caddyfile` is configured to proxy `app.lickity.click` to port `3003` and short domains to port `8081`.
</Callout>
